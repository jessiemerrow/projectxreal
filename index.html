A) Requirements - top

This application is developed with one of the latest versions of Laravel 8, one of the safest and most used frameworks. To successfully install this script, make sure your hosting plan meets the minimum requirements:

  • PHP >= 7.3

B) Installation - top

The .zip file contains 3 folders, "Upload", "Database" and "Documentation".

1) Move the content of "Upload" folder to your domain name folder.

Note: Depending on the file transfer you will be using, it may not fully transfer all files.
So make sure that any files, such as .htaccess, are present.

2) Database

Here, you should already have a database created in your server, in order to do that (if it's not already done), in yout server cPanel, click on "MySQLÂ® Databases"

then create a new user for your database:

and finally add the user you've just created to your database

Note: the user's username & password and the database's name are what you need to configure your database

2. a) PHPMyAdmin

Now you need to connect to the newly created database via PHPMyAdmin that you find from your hosting panel options.

Then select your database and go to the "Import" menu item to select the .sql file that you find in the "Database" folder.

Then select one of the two files (one for older versions of MySQL, the other for the most recent), and run.

Note: if the folders were not created, and there were errors, alternate importing the other database.

All tables have now been created, the database is ready. Continue to connect your script to the database.

C) Mail - top

The mailer is required to send and receive email messages, for support, orders ... etc.

  • In your server's cPanel, search/click on "Email Accounts"



  • Then click on "Connect Devices" for your default email account or create a new one



  • Then, at the bottom, you have two choices, secure and none-secure configuration (settings), the password is the one you used to create the new email account or the cPanel one if it's the default email account.



Open the .env file again to insert the newly created SMTP parameters:




Assuming you entered the correct data, save the file again and that's it. Now everything is ready and you can test your site by reaching it at your url address.

By default there is an administrator account, we advise you not to change the role if it is the only administrator user present.

You can connect with the administrator account already present by default, log in with the following data:

E) Advertising - top

To create a new ad unit, you need to log into your admin area.

            Dashboard / Settings / Advertising

Manage ad units or create new ones.
To set which one to make visible in your application, go to the settings and in the block called "Advertising", at the bottom of the page, select which unit to make visible.
